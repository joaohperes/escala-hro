# Arquivo

Material histórico do projeto — mantido para consulta, **não usado em produção**.

- `docs/` — registros de correções, análises e guias antigos (a documentação viva é `README.md`, `ARCHITECTURE.md` e `MAINTENANCE_CHECKLIST.md` na raiz).
- `scripts/` — scripts pontuais já executados (coleta de contatos em lote, correções únicas, debug). Podem ter caminhos desatualizados.
- JSONs soltos — dados intermediários de coletas antigas de contatos.
